## Prerequisites
Integrate SCIM With <BrandName />:

- You will need an Enterprise plan with <BrandName />.
- SSO must be already integrated. Please complete [<BrandName /> SSO & Okta Integration](/support/docs/testmu-sso-okta-integration/)

## Integrating SCIM with Okta
**Step 1:** Sign in to your <BrandName /> account. Don't have an account, [register for free](https://accounts.lambdatest.com/register).

<img loading="lazy" src='/assets/images/testmu ai-mfa/dashboard.webp' alt="Image" width="404" height="206"  className="doc_img img_center" /><br />

**Step 2:**  Head to **Settings** and select **Organization Settings** from the dropdown.

<img loading="lazy" src='/assets/images/testmu ai-mfa/org-settings.webp' alt="Image" width="404" height="206"  className="doc_img img_center" /><br />

**Step 3:**  Head to the **Authentication and SSO** tab and click and copy the **SCIM Base URL and Bearer Token** option.

<img loading="lazy" src='/assets/images/testmu ai-scim/scim-base-url.png' alt="Image" width="404" height="206"  className="doc_img img_center" /><br />

**Step 4:** Sign in to the [Okta](https://developer.okta.com). Select Enterprise Applications, then select Applications tab.
<img loading="lazy" src='/assets/images/testmu ai-scim/okta/applications-tab.png' alt="Image" width="404" height="206"  className="doc_img img_center" /><br />

**Step 5:** In the applications list, select application used for <BrandName /> SSO setup

**Step 6:** Select the Provisioning tab.
<img loading="lazy" src='/assets/images/testmu ai-scim/okta/provisioning-tab.png' alt="Image" width="404" height="206"  className="doc_img img_center" /><br />

**Step 7:** Under the Integration section, input your <BrandName /> SCIM Base URL-`https://auth.lambdatest.com/api/scim`  and Bearer Token retrieved earlier in Step 3. 

**Unique identifier field for users** field must be set to **userName**
Click Test Connection to ensure Okta can connect to <BrandName />. If the connection fails, ensure your <BrandName /> account has Admin permissions and try again.
<img loading="lazy" src='/assets/images/testmu ai-scim/okta/integration-tab.png' alt="Image" width="404" height="206"  className="doc_img img_center" /><br />

**Step 8:** Now navigate to App tab
<img loading="lazy" src='/assets/images/testmu ai-scim/okta/app-tab.png' alt="Image" width="404" height="206"  className="doc_img img_center" /><br />

**Step 9:** Review the User Attribute mappings:
<img loading="lazy" src='/assets/images/testmu ai-scim/okta/mapping-tab.png' alt="Image" width="404" height="206"  className="doc_img img_center" /><br />

userName, Active, name.givenName, name.familyName are required attributes

**Creating Custom Attributes**

For creating custom attribute in Okta, go to Directory > Profile Editor > Add Attribute > [Create Custom Attribute](https://help.okta.com/en-us/content/topics/users-groups-profiles/usgp-add-custom-user-attributes.htm)

**`urn:ietf:params:scim:schemas:extension:TestMu AI:2.0:User:OrganizationRole`**: 

**`urn:ietf:params:scim:schemas:extension:TestMu AI:2.0:User:LambdatestGroup`** : Applicable only if organisation has group support active)

Enter Display Name as per your choice 
Enter Variable name as **OrganizationRole**
Enter External name as **OrganizationRole**
Enter External namespace as **urn:ietf:params:scim:schemas:extension:<BrandName />:2.0:User**
<img loading="lazy" src='/assets/images/sso/img_18.png' alt="okta integration" width="944" height="487" />

Set the data type as **String**
Select Enum as **Yes**
Enter the values as **Admin, Guest, User**
<img loading="lazy" src='/assets/images/sso/img_19.png' alt="okta integration" width="944" height="487" />

Also if you want to assign this attribute at a okta group level choose AttributeType as Group

<img loading="lazy" src='/assets/images/sso/img_20.png' alt="okta integration" width="944" height="487" />

**Step 10:** To enable the Okta provisioning service for <BrandName />, set Create Users, Update User Attributes and Deactivate Users to enabled
<img loading="lazy" src='/assets/images/testmu ai-scim/okta/provisioning-enabled.png' alt="Image" width="404" height="206"  className="doc_img img_center" /><br />
