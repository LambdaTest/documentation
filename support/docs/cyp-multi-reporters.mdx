With <BrandName />, you can generate multiple reporters supported by Cypress for your tests. If you want to do so, follow the steps below and download the generated reports automatically.

***

#### Step 1

Provide the path where your report artefacts will be generated by the test script in the `testmu ai-config.json` file. In the same file, add the path of the reporter config file that contains the configurations that your reporters require. 

The example `testmu ai-config.json` file2 shown below uses 2 reporters, `mochawesome` and `mocha-junit-reporter`.

```bash
"run_settings":{
.....
"reporter_config_file": "reporter-config_mochawesome_junit.json",
"downloads": "./cypress/results",
.....
}
```

***

#### Step 2

Create a file with the name `reporter-config_mochawesome_junit.json` and copy the contents of the code block below. 

```bash
{   
    "reporterEnabled": "mochawesome,mocha-junit-reporter",
    "mochawesomeReporterOptions": {
        "reportDir": "cypress/results/json",
        "overwrite": true,
        "html": true,
		"json": true
    },
    "mochaJunitReporterReporterOptions": {
        "mochaFile": "cypress/results/my-test-output.xml",
        "toConsole": true
    }
  }
```

***

#### Step 3

Run your test in sync mode by using the code `testmu ai-cypress run --sync=true`.

After all the tests are run, the generated reports will be automatically downloaded at the location `testmu ai-artefacts`. 

<Info>
****Note****

The location of `testmu ai-artefacts` is relative to the path from where you ran the command.
</Info>
