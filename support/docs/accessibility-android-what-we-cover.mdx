**Technology:** <BrandName /> Accessibility for **native Android apps** (Views, Jetpack Compose where applicable). **Platform:** Android (check <BrandName /> docs for supported API levels and devices).

## 3. What we cover – checklist (Android)

### 3.1 WCAG 2.0 (Level A and AA) – Android-relevant

| WCAG | Level | Brief description | What we cover (Android) |
|------|-------|-------------------|--------------------------|
| 1.1.1 Non-text Content | A | Text alternatives for non-text content | Images have `android:contentDescription`. |
| 1.3.1 Info and Relationships | A | Structure programmatically determinable | Buttons, checkboxes, toggles have names and roles. |
| 1.4.3 Contrast (Minimum) | AA | Text contrast 4.5:1 (3:1 large) | Insufficient color ratio detection. |
| 2.1.1 Keyboard | A | All functionality via keyboard/AT | Operable via TalkBack and switch access. |
| 2.1.2 No Keyboard Trap | A | Focus not trapped | Focus can leave modals/custom controls. |
| 2.4.2 Page Titled | A | Screens have titles | Screen/activity titles. |
| 2.4.3 Focus Order | A | Focus order preserves meaning | Logical focus order; no nested interactive elements. |
| 3.3.2 Labels or Instructions | A | Labels provided | Field labels (hint, labelFor, TextInputLayout). |
| 4.1.2 Name, Role, Value | A | Name, role, value for UI components | contentDescription, roles, values; unique labels. |

## 5. How to check and how to do (Android)

### 5.1 Labels (1.1.1, 1.3.1, 3.3.2, 4.1.2)

- **Images:** Set `android:contentDescription` or `null` for decorative.
- **Buttons:** Set `contentDescription` or `android:text`.
- **EditText:** Use `android:hint`, `labelFor`, or TextInputLayout.
- **Switches:** Set `contentDescription` describing what is toggled.

### 5.2 Color contrast (1.4.3)

- Use colors meeting **4.5:1** for normal text, **3:1** for large text.
- Use theme colors (`?attr/colorOnSurface`, `?attr/colorSurface`).

### 5.3 Touch target sizing (2.5.5, 2.5.8)

- Ensure **48dp × 48dp** minimum with adequate spacing.

### 5.4 Orientation (1.3.4)

- Use `android:screenOrientation="unspecified"` unless lock is essential.
